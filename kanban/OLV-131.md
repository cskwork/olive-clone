## Why

Standard for any container deploy. Liveness must stay cheap; readiness
must verify downstreams.

## Scope

- `/actuator/health` (liveness): default Spring `HealthIndicator` —
  cheap, no downstream calls.
- `/actuator/health/readiness` (readiness): custom composite that
  checks Postgres (`SELECT 1`), Redis (`PING`), OpenSearch
  (`/_cluster/health` ≠ red).
- Custom `BatchHealthIndicator` reports the count of `outbox_events`
  with status DEAD as a degraded signal.
- Document the contract in `README.md`: liveness = process alive,
  readiness = traffic-ready.

## Acceptance Criteria

- [ ] `/actuator/health` returns 200 / `{"status":"UP"}` even if Redis
      is stopped.
- [ ] `/actuator/health/readiness` returns 503 when Redis or OpenSearch
      is down, 200 otherwise.
- [ ] Stopping Postgres makes BOTH endpoints return 503 (DB outage is
      total).
- [ ] Tests use Testcontainers to flip dependencies and verify each
      response code.
