## Why

Per-ticket QA proves each module works in isolation. This ticket proves
they compose into the purchase flow from PRD §9.1 end-to-end.

## Scope

A single `PurchaseFlowE2ETest` (Spring Boot Test + Testcontainers Postgres
+ Redis + OpenSearch + LocalStack) that walks PRD §9.1 exactly:

1. POST signup + login → bearer token.
2. Admin token: create brand → category → product (2 options) → upload
   image presigned → restock 100 each.
3. User token: GET `/api/products` → finds product. GET detail.
4. POST `/api/cart/items` (option A, qty 2).
5. POST `/api/orders` with cart items + couponId (admin pre-issued) +
   usePointAmount (admin pre-credited).
6. POST `/api/payments/confirm` with `X-Mock-Pg-Behaviour: approve`.
7. Wait up to 5s for `outbox_events` of PAYMENT_APPROVED to be DONE.
8. Assert order status PAID, inventory committed (-2 from total),
   `points` USE row written, `member_coupons.status=USED`.
9. POST a webhook `DELIVERY_COMPLETED` (or fast-forward delivery
   scheduler).
10. Assert order status DELIVERED, `points` EARN flipped to spendable,
    review-eligibility cache populated.
11. POST a review.
12. Assert `product_review_summaries.review_count = 1`.

Capture artefacts under `docs/OLV-140/qa/`:
- `flow.har` of every step's request/response (use HTTP recorder).
- `final-state.json` dumping the relevant DB rows.

## Acceptance Criteria

- [ ] Test passes from a clean DB in <60s.
- [ ] Replaying step 6 with the same idempotency key yields identical
      state (no duplicate inventory commit).
- [ ] Negative branch: a copy of the test that uses
      `X-Mock-Pg-Behaviour: fail` ends with order in PAYMENT_PENDING,
      reservation still HELD, no outbox PAYMENT_APPROVED row.

## Hints

- Use `@DynamicPropertySource` to wire Testcontainers ports into Spring
  config.
- For the delivery flip, prefer calling the scheduled bean directly via
  `@Autowired` rather than waiting wall-clock time.

**핵심 도메인 흐름**:
```
회원가입 → 상품생성(Admin) → 장바구니 → 주문생성 → 결제승인 → 배송완료 → 리뷰작성
```

- **재고**: reserve-then-commit (주문 시 선점, 결제 시 확정)

- **배송**: DeliveryCompletedEvent → 포인트 spendable 전환 + 리뷰 가능 표시

## Learnings

1. **트랜잭션 격리**: `@BeforeEach`에서 `txTemplate.executeWithoutResult()` 사용 시 테스트 메서드에서 데이터를 조회하지 못하는 문제 발생. `jdbcTemplate` 직접 사용으로 해결.
2. **리뷰 자격 검증**: `ReviewRepository.isEligibleForReview()`는 `orders.status = 'DELIVERED'`를 확인하지만, 배송 완료 처리는 `deliveries.status`만 변경. 테스트에서 주문 상태도 명시적으로 변경 필요.
3. **비동기 이벤트 리스너**: `@Async` + `@TransactionalEventListener`는 테스트 환경에서 즉시 실행되지 않아 `await` 패턴 또는 직접 INSERT로 우회 필요.
4. **프로덕션 버그 발견**: `PaymentService`에서 `payment_transactions` 기록 실패 ("Failed to record transaction for payment 1: null") - 별도 티켓에서 수정 필요.
