## Why

PRD §6.3, §8.1: this is the user-facing search endpoint. Without it,
discovery is limited to category browsing.

## Scope

Endpoints under `/api/search`:

- `GET /api/search/products?keyword=&categoryId=&brandId=&sort=&page=&size=`
  — full-text search against the OpenSearch index. Response shape mirrors
  `/api/products` for client reuse. Sort options: relevance (default),
  popular (salesCount desc), latest (createdAt desc), price asc/desc,
  rating desc.
- `GET /api/search/autocomplete?prefix=&size=10` — prefix completion on
  productName + tags + brandName. Cached in Redis 5 min by prefix.
- `GET /api/search/popular?size=10` — popular search keywords from the
  last hour, computed by a `@Scheduled` job that aggregates a Redis ZSET
  written by `/api/search/products`.

Failure handling per `llm-wiki/99-failure-handling.md`: if OpenSearch is
down, return 503 with `{"error":"검색 일시 중단"}` — never silently fall
back to DB LIKE.

## Acceptance Criteria

- [ ] Search `keyword=선크림` returns all ON_SALE products whose name or
      tags contain "선크림", sorted by relevance.
- [ ] Filter by categoryId combined with keyword returns the intersection.
- [ ] Stopping OpenSearch returns 503 with the documented body.
- [ ] Autocomplete returns prefix matches (case-insensitive).
- [ ] Popular keywords list is non-empty after seeding 100 fake searches
      via the test harness.
