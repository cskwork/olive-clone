## Why

PRD §17. Without these, expired reservations leak inventory, expired
coupons stay clickable, and revenue dashboards stop updating.

## Scope

Implement each as a `@Scheduled` bean with `ShedLock` for multi-instance
safety. Each job writes a `job_runs` row.

- `PaymentPendingExpiryJob` (every 5 min): finds orders in
  `PAYMENT_PENDING` for > 30 min. For each, `verify(paymentKey)` against
  PG; if PG says FAIL/UNKNOWN → set order `FAILED`, release reservation,
  restore coupon (do NOT restore points used because they weren't used
  yet — OLV-072's point.use only runs on PAID).
- `InventoryReservationExpiryJob` (every 5 min): finds reservations
  HELD with `expires_at < now()`. Releases each. Writes
  `inventory_histories`.
- `CouponExpiryJob` (daily 00:00): flips `member_coupons` to EXPIRED
  where `expires_at < now()` and status = ISSUED.
- `PointExpiryJob` (daily 00:00): writes EXPIRE rows for unused EARN with
  `expires_at < now()`.
- `DeliveryStatusSyncJob` (every 10 min): walks active deliveries, calls
  `CarrierClient.fetchStatus`, applies status transitions. Failures →
  `delivery_retry_queue`.
- `SalesAggregationJob` (daily 02:00): computes daily revenue by
  category/brand/product, writes `daily_sales_summaries` (table created
  in this migration).
- `ProductRankingJob` (every 1 hour): recomputes top-100 products by
  weighted (salesCount × 0.5 + reviewCount × 0.3 + avg_rating × 0.2)
  and writes `product_rankings` (table created in this migration).

Migration `V12__batch_tables.sql`: `job_runs`, `daily_sales_summaries`,
`product_rankings`.

Admin endpoint `POST /api/admin/jobs/{jobName}/run-now` for manual replay
(requires `SUPER_ADMIN`).

## Acceptance Criteria

- [ ] Test harness fast-forwards the clock; each job processes the
      expected rows.
- [ ] ShedLock prevents two app instances from running the same job
      simultaneously (verify by manual inspection or via the lock
      provider's logs).
- [ ] Manual run endpoint triggers the job once even if the next
      scheduled tick hasn't arrived.
