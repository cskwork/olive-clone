## Why

Cache, distributed lock, search, and image upload tickets all reference
infra that must already exist. Adding it once now saves five repeat docker
edits later.

## Scope

Extend `docker-compose.yml` with:

- `redis:7-alpine` (port 6379, no auth in dev).
- `localstack/localstack:latest` (port 4566) preconfigured with one
  S3 bucket `commerce-images-local`.
- `opensearchproject/opensearch:2` (port 9200) single-node, security plugin
  disabled in dev (`DISABLE_SECURITY_PLUGIN=true`).

Wire Spring config for `local` profile:

- `spring.data.redis.host/port` for Redis.
- `aws.s3.endpoint=http://localhost:4566`, region `ap-northeast-2`, signing
  with anonymous-equivalent credentials suitable for LocalStack.
- `opensearch.uris=http://localhost:9200`.

Add a `common.config` module with one bean per integration
(`RedisTemplate<String,String>`, `S3Client`, `OpenSearchClient`) so domain
tickets just `@Autowired` them.

## Acceptance Criteria

- [ ] `docker compose up -d` brings all four services up.
- [ ] `./gradlew test` includes one Testcontainers integration test per
      service that proves the bean is healthy: Redis SET/GET, S3 PUT/GET,
      OpenSearch index existence check.
- [ ] Boot logs show all three clients initialized at startup with no
      stack traces.

## Hints

- Use `org.springframework.boot:spring-boot-starter-data-redis`
  (Lettuce client by default).
- For S3, use the AWS SDK v2 (`software.amazon.awssdk:s3`).
- For OpenSearch, use the official `opensearch-rest-client` +
  `opensearch-java` library.
- Add Testcontainers modules: `redis`, `localstack`, `opensearch`.
