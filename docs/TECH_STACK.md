# Tech Stack

This project is built to look and behave like a production commerce backend while
remaining runnable on a local laptop.

## Stack Summary

| Layer | Main choices | Project role |
| --- | --- | --- |
| Language | Java 21 | LTS runtime baseline and modern language features. |
| Framework | Spring Boot 3.3 | Application bootstrapping, web APIs, security, validation, metrics, tests. |
| Build | Gradle Kotlin DSL | Wrapper-based builds, dependency management, custom BootRun tasks. |
| UI | Thymeleaf, static CSS, vanilla JavaScript | Small server-rendered smoke console that exercises the real product API. |
| Database | PostgreSQL 16 | Source of truth for commerce data and operational audit tables. |
| Migration | Flyway 10 | Repeatable local setup through ordered schema and seed migrations. |
| ORM | Spring Data JPA, Hibernate | Repository layer and entity mapping. |
| Cache | Redis | Public catalog caching and supporting infrastructure. |
| Locks | Redisson, ShedLock JDBC provider | Inventory/scheduler coordination and distributed job safety. |
| Search | OpenSearch Java client | Product search, autocomplete, popular keywords, and full reindexing. |
| Files | AWS SDK S3 client, LocalStack | S3-compatible image/upload flows in local development. |
| Observability | Actuator, Micrometer, Prometheus, Grafana | Health, metrics, dashboard, and alerting assets. |
| Tests | JUnit 5, MockMvc, Spring Boot Test, Testcontainers | Fast controller/service checks and real-infra integration tests. |
| Load tests | k6 | Scripted product and order traffic. |

## Application Runtime and Build

- **Java 21**: modern LTS runtime and language baseline.
- **Spring Boot 3.3**: web, configuration, validation, security, observability,
  and testing ecosystem.
- **Gradle wrapper + Kotlin DSL**: reproducible local builds without requiring a
  global Gradle installation.
- **Dependency management**: Spring dependency management plugin, Testcontainers
  BOM, and AWS SDK BOM keep framework and infrastructure libraries aligned.
- **Custom run task**: `reindexProducts` boots the app with the `reindex`
  profile and rebuilds OpenSearch documents from Postgres.

## Web and Security

- **Spring MVC** serves REST APIs and the small Thymeleaf smoke UI.
- **Thymeleaf** is used for the `/products` catalog console. The page loads data
  from the public product API, so it exercises the same API contract as a client.
- **Static CSS and vanilla JavaScript** keep the smoke UI dependency-free and
  easy to inspect from a fresh clone.
- **Bean Validation** guards request DTOs before service logic runs.
- **Spring Security OAuth2 Resource Server** validates JWT-protected endpoints.
- **RS256 JWT** access and refresh tokens model production-style signing and
  verification. Local signing keys are generated by the developer and ignored by
  Git.
- **Nimbus JOSE JWT** provides lower-level JWT support where the application
  needs explicit token creation and signing behavior.

## Data and Storage

- **PostgreSQL 16** is the source of truth.
- **Flyway 10** owns schema migration and demo seed data. Public-demo migrations
  seed 13 products and map their thumbnails to local static assets.
- **Spring Data JPA / Hibernate** handles repository access and entity mapping.
- **Redis** backs public catalog caching and distributed lock infrastructure.
- **Redisson** provides lock primitives for inventory and scheduled work.
- **ShedLock JDBC provider** protects scheduled jobs from duplicate execution
  when multiple app instances are running.
- **LocalStack S3** supplies a local S3-compatible endpoint for image upload
  flows.
- **Static product assets** under `src/main/resources/static/images/products`
  keep the seeded catalog visually complete in a fresh local demo.

## Search and Async Work

- **OpenSearch** stores product search documents.
- **OpenSearch Java client and REST client** are used for index management,
  bulk indexing, search, autocomplete, and popular keyword queries.
- **Outbox table** records index-sync events transactionally beside product data.
- **Indexer worker** drains outbox events and retries failures without losing the
  product source of truth.
- **Reindex command** (`./gradlew reindexProducts`) hydrates all products from
  Postgres and rebuilds OpenSearch, which keeps local demos recoverable.

## Operations

- **Spring Boot Actuator** exposes liveness, readiness, and Prometheus metrics.
- **Micrometer Prometheus registry** exports metric data at
  `/actuator/prometheus`.
- **logstash-logback-encoder** supports structured JSON-friendly logging with
  trace IDs.
- **Prometheus and Grafana** are provisioned through Docker Compose and checked-in
  dashboard files.
- **Alert rules** live under `infra/prometheus/alerts.yml`.
- **k6** scripts cover product-list and order-create load scenarios.

## Testing

- **JUnit 5 and MockMvc** cover controller and service behavior.
- **Spring Security Test** covers authenticated and role-protected paths.
- **Testcontainers** provides real Postgres, Redis, LocalStack, and OpenSearch
  services during integration tests.
- **Flyway validation** guards migration drift.

## Local Infrastructure

Docker Compose provides all external services needed for the demo:

| Service | Local purpose |
| --- | --- |
| Postgres | Primary relational database and Flyway migration target. |
| Redis | Cache and lock backend. |
| LocalStack | S3-compatible local object storage. |
| OpenSearch | Search index and autocomplete/popular keyword backend. |
| Prometheus | Metric scraping for the app. |
| Grafana | Dashboard visualization from checked-in provisioning files. |
