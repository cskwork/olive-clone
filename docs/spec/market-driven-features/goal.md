# 시장 조사 기반 신규 기능 후보 — Goal / Vision

> **무엇:** 올리브영형 H&B 커머스(현 `com.olive.commerce` 백엔드 + React 스토어프론트)에 더할 **net-new 기능 후보 포트폴리오**를 시장 조사 근거와 함께 우선순위로 정리한 goal 수준 문서.
> **형식:** supergoal SPEC 모드 / **goal-only** — 여기서는 *무엇을 왜* 만들지까지만 다룬다. 구현 설계(design.md)와 태스크(tasks.md)는 의도적으로 제외.
> **작성:** 2026-06-20 · **조사 방식:** 실시간 웹 조사(4개 스트림, WebSearch/WebFetch) · **시점 기준:** 2025-2026 우선
> **면책:** 본 프로젝트는 교육용 포트폴리오 클론으로, 올리브영/특정 브랜드와 무관하다. 아래 경쟁사 기능은 *벤치마크 신호*로만 인용한다. 시장 규모 수치는 출처별 편차가 커 방향성 참고용이다(자세한 caveat는 §9).

## 1. 목표 (Goal)

현 프로젝트는 카탈로그·장바구니·주문·(mock)결제·검색·랭킹·기간한정딜·구매검증 리뷰·위시리스트·마이페이지·인증까지 **핵심 커머스 흐름이 구현된 상태**다. 따라서 다음 한 줄이 이 문서의 목표다.

> **"이미 작동하는 커머스 골격 위에, 2025-2026 K-뷰티/H&B 시장이 실제로 요구하는 차별화 기능을 얹어 '범용 쇼핑몰'을 '뷰티 전용 몰'로 끌어올린다."**

핵심 전환축 세 가지:
1. **범용 카탈로그 → 피부 맞춤 개인화** (내 피부/고민 중심 탐색·리뷰·추천)
2. **단순 상품 정보 → 성분·정품·가격 신뢰 레이어** (K-뷰티 구매자가 실제로 검증하는 축)
3. **검색/브라우즈 → 콘텐츠·라이브·빠른배송 경험** (발견과 전환을 끌어올리는 채널)

## 2. 현재 위치 & 갭 (확인된 사실)

`src/main/java/com/olive/commerce` 직접 확인 결과, 아래 제안 기능은 **코드베이스에 전혀 없음**(net-new 확정):
`skin profile · ingredient/allergen · live commerce · subscription/recurring · gift order · price history · virtual try-on` → grep 0건.

| 이미 있음 (재사용 자산) | 없음 (이 문서의 후보) |
|---|---|
| 구매검증 리뷰, 위시리스트, 알림(NotificationService), 주문/배송, 쿠폰/적립금, 랭킹(ProductRanking), SalesAggregator, 검색+자동완성 | 피부 프로파일, 성분 DB/필터, 라이브/숏폼, 빠른배송·매장재고, 등급제 멤버십, 가격추적, 정품보증, 선물하기, AR/AI 어시스턴트, 정기배송 |

→ 신규 후보 대부분이 **기존 자산(리뷰·위시리스트·알림·주문·쿠폰)을 재사용**해 증분 구축 가능하다는 점이 우선순위의 핵심 근거다.

- **가장 강한 1차(first-party) 신호 (가장 신뢰도 높음):**

## 4. 신규 기능 후보 포트폴리오

티어 = **수요 × (기존 자산) 재사용성/실현성 × 차별화**. 각 후보: JTBD(목표 사용자/할 일) · 시장 근거 · MVP 범위 · 가장 위험한 가정 · 재사용/노력도.

### Tier 1 — 지금 (저비용·in-house·개인화 기반축, 서로 강화)

#### MDF-01 · 피부 프로파일 & 맞춤 탐색 (Skin Profile & Personalized Discovery)
- **JTBD:** *"내 피부/고민에 맞는 제품을 빠르게 찾고 싶다."* — 모든 뷰티 구매의 핵심 작업.
- **시장 근거:** 올리브영 SkinScan(100만 건, 전환 73% vs 53%), 화해 '내 피부 맞춤'+궁합 점수, 무신사 'My Skin'; 효능이 구매 1순위(38.7~53%). 4개 조사 스트림 모두에서 1순위로 수렴.
- **MVP:** 가입/마이페이지의 **경량 진단 퀴즈**(피부타입·고민·연령대·민감도) → `SkinProfile` 객체 → 홈 '내 피부 맞춤' rail + PDP "내 피부에 맞아요" 태그 + 고민별 내비게이션. **카메라/CV 불필요**(셀피 진단은 Watchlist W1로 분리).
- **가장 위험한 가정:** 사용자가 퀴즈를 완료할 만큼 가치를 느낀다(cold-start). 미완료 시 기존 일반 rail로 graceful fallback.
- **재사용/노력도:** 신규 프로파일 도메인 + 기존 랭킹/추천에 boost. **낮음-중간.** 다른 모든 개인화 후보의 substrate.
- **가드레일:** 피부 데이터 = PIPA상 민감정보 → 파생 속성만 저장, 명시 동의, 보존기한(§6).

#### MDF-02 · 내 피부 맞춤 리뷰 ("나와 같은 피부" 리뷰)
- **JTBD:** *"이 4.8점 제품이 '내 피부'에도 맞나?"* — 평균 별점은 건성-50대에겐 오해를 부른다.
- **시장 근거:** 화해/글로우픽/무신사/Sephora/Ulta 표준 기능. 리뷰 = #1 정보 니즈(70%). 더불어 **포토/영상 리뷰 + 부정리뷰 비은폐 + 가짜리뷰 플래그**는 공정위 후기조작 제재(2025)라는 규제 순풍.
- **MVP:** 리뷰 작성 시 작성자 피부속성 태깅 → PDP 리뷰를 "나와 같은 피부"로 **필터/정렬** + 매칭 작성자만의 **보조 평점**. 포토/영상 업로드 + 정렬(도움순). **완전 in-house, 기존 구매검증 리뷰 확장.**
- **가장 위험한 가정:** 세그먼트별 리뷰 수가 충분(희소 세그먼트는 매칭 평점 숨김 처리). MDF-01 프로파일에 의존.
- **재사용/노력도:** 기존 리뷰 스키마 확장. **낮음.** 본 포트폴리오에서 **가성비 1위**.

#### MDF-03 · 성분 인텔리전스 (전성분 분석 + 알레르기/성분 필터)
- **JTBD:** *"성분을 확인하고, 내가 피하는 성분이 든 제품을 거르고 싶다."*
- **시장 근거:** 화해 ~1,000만 사용자(클론의 최대 공백), Sephora 'Clean' 배지, 컬리 전성분 의무공개; 성분 중요도는 연령과 함께 상승(40대 28%). 비건/클린뷰티 트렌드와 결합.
- **MVP:** 상품별 **전성분(INCI) 패널** + EWG/주의·알레르기 성분 플래그 + 성분 사전 + **성분 포함/제외 검색** + 개인 'avoid' 워치리스트 경고 + 비건/클린/저자극 facet·배지.
- **가장 위험한 가정:** 신뢰할 성분 마스터 데이터를 확보/라이선스할 수 있다(가장 어려운 부분). EWG는 **규제기관 공인 아님 → '의학적 사실' 아닌 '가이드'로 표기**(§6).
- **재사용/노력도:** 신규 성분 도메인 + 검색 facet 재사용. **중간**(데이터 소싱이 관건), 효과 큼.

### Tier 2 — 다음 (높은 가치·중간 빌드, 대부분 기존 자산 재사용)

#### MDF-04 · 신뢰 & 가격 투명성 (정품 보증 + 가격 추적/인하 알림)
- **JTBD:** *"정품인지 믿고, 가장 쌀 때 사고 싶다."*
- **시장 근거:** 가품 447건(향수 51.5%, 온라인 70.7%) + 공정위 압박 → 정품 신뢰가 마켓플레이스 대비 차별점. 플랫폼 간 가격차 20.9%, 가격추적 앱(폴센트/에누리)이 별도 카테고리로 존재 = **경쟁사가 약한 차별화 지점**.
- **MVP:** PDP **정품/공식채널 배지** + 배치/시리얼·유통기한 표기 + '가품 의심·기한임박 신고' 플로우(→admin); **가격 히스토리 그래프** + "최근 30/90일 최저" 배지 + 위시리스트 **목표가/인하 알림**(기존 위시리스트+알림 재사용).
- **가장 위험한 가정:** 가격 히스토리는 데이터 축적이 필요(초기엔 빈약). 1st-party이므로 정품은 주로 신뢰 시그널링.
- **재사용/노력도:** 위시리스트+NotificationService 재사용. **중간.** 신뢰 차별화.

#### MDF-05 · 로열티 & 멤버십 엔진 (등급제 + 월 재산정 + 리워드 교환)
- **JTBD:** *"빠르게 반응하고 머물 가치가 있는 보상."*
- **시장 근거:** 올리브영 5등급 **월 재산정**(1,600만), Sephora Beauty Insider + Rewards Bazaar(포인트→체험 교환), 컬리멤버스 유료 구독(₩1,900/월). 올영세일/올영데이/어워즈 = **쿠폰 스태킹 + 데이터 기반 큐레이션**.
- **MVP:** 누적지출 기반 **등급 산정(롤링 윈도, 월 재계산)** + '다음 등급까지' 진행바 + 등급쿠폰 자동지급 + 쿠폰 스태킹 규칙. (유료 구독 등급은 실결제 의존 → 후순위.)
- **가장 위험한 가정:** 대부분 백엔드 계산 — 리워드 카탈로그 운영 리소스가 필요.
- **재사용/노력도:** 기존 적립금/쿠폰/주문 집계 확장. **낮음-중간.** 평탄한 포인트 → 리텐션 엔진으로 승급.

#### MDF-06 · 선물하기 & 소셜 구매 (Gifting + 안전한 공동구매)
- **JTBD:** *"주소 없이 연락처로 뷰티를 선물하고, 같이 안전하게 사고 싶다."*
- **시장 근거:** 올리브영 선물하기(연락처 전송, 수령자가 색상/향 변경), 화장품 = 인스타 공구 1위 품목이나 **사기 만연(₩9,600만 피해)** → '안전한 1st-party 공동구매' 수요 미충족.
- **MVP:** **연락처/카카오 기반 선물하기**(전화번호 전송, 카드 메시지, 수령자가 옵션 선택+본인 주소 입력, 선물상태 추적) — 기존 checkout/주소록 재사용. *스트레치:* 정품보증 그룹구매/공유장바구니.
- **가장 위험한 가정:** 선물하기는 기대 기본값(낮은 신규성)이나 전환·바이럴 클레임 링크 효과 큼. 그룹구매는 운영 부담.
- **재사용/노력도:** checkout/order/주소록 재사용. 선물하기 **낮음**, 그룹구매 **중간**.

### Tier 3 — 전략적·중량급 (운영/인프라 의존, 효과는 최상위)

#### MDF-07 · 빠른배송 & 매장 풀필먼트 (오늘드림형 + 매장재고/픽업)
- **JTBD:** *"오늘 안에 받거나, 가까운 매장에서 바로 픽업하고 싶다."* — 한국 H&B의 정의적 행동.
- **시장 근거:** 오늘드림 매출 비중 ~28-30%, 이용률 48%; 컬리/신세계/쿠팡 등 업계 전체가 1시간 배송 경쟁. **통증점:** 무리한 당일 접수 후 22시 자동취소·통보 부실(컨슈머머치 2026-03).
- **MVP:** 체크아웃 **당일/익스프레스 옵션** + PDP **매장/MFC 재고 표시** + **BOPIS(예약·픽업)** + **정직한 ETA: 실시간 슬롯 용량 → 불가 시 선차단(접수 후 취소 금지)** + 상태 타임라인 푸시 + 대체배송 fallback.
- **가장 위험한 가정:** 실 매장 재고/물류가 필요(포트폴리오에선 일부 mock 가능). 순수 소프트웨어가 아닌 운영 의존.
- **재사용/노력도:** 신규 store/inventory 모델 + 배송슬롯. **중간-높음.** 전략적 핵심.

#### MDF-08 · 라이브 & 콘텐츠 커머스 (라이브 + 매거진 + 숏폼 UGC 피드)
- **JTBD:** 검색/브라우즈가 아니라 **영상·콘텐츠로 발견**하고 싶다.
- **시장 근거:** 올영라이브 전환 22%/최다 73만, 매거진 1,070만 뷰, 쇼핑 큐레이터 7% 커미션·셔터 UGC 피드; TikTok Shop = 미국 소셜커머스 18.2%, UGC 광고 CTR 4배.
- **MVP:** **쇼퍼블 VOD**(상품 carousel 동기화) + 예약 라이브(라이브 전용 쿠폰·채팅) + **인앱 매거진**(콘텐츠→상품 딥링크) + 리뷰 파이프라인 기반 숏폼 피드 + 리뷰어 등급/팔로우. (실시간 라이브 인프라는 무거우니 **VOD부터**.)
- **가장 위험한 가정:** 콘텐츠 제작/운영 리소스. 진성 라이브 스트리밍은 중량급 → 단계적.
- **재사용/노력도:** 리뷰 파이프라인 + 기간한정딜/쿠폰 엔진 재사용; CMS 신규. **중간-높음.**

## 6. 교차 리스크 & 가드레일 (착수 전 필독)

- **개인정보(PIPA):** 피부 프로파일/셀피 파생 데이터는 민감정보. **원본 이미지 비저장**(벤더측 처리 선호), 파생 점수만 저장, **명시 동의 + 보존기한** 설정. (MDF-01, W1, W3)
- **성분 정보 책임:** EWG 등급은 규제기관 공인이 아님 → **"참고용 가이드"로 표기, 의학적/효능 단정 금지**. (MDF-03)
- **표시광고 규제:** 2024H2~2025H1 화장품 행정처분의 76%가 표시·광고 위반(의약품 오인 등). 카피·배지에 **claim 컴플라이언스 가드** 필요. (MDF-03, MDF-04)
- **후기조작 규제:** 공정위가 부정리뷰 은폐·뒷광고 제재 중 → **부정리뷰 비은폐 정책 + 유료 리뷰 표시**. (MDF-02)
- **결제 의존:** 정기배송(W2)·유료구독 등급(MDF-05)은 **실 반복결제 PG 전제** — 현 mock 결제로는 불가, 기존 Phase 5 실 PG 연동에 종속.
- **운영 의존:** MDF-07(빠른배송)·MDF-08(라이브)은 순수 소프트웨어가 아닌 **물류/콘텐츠 운영**이 성패를 가름. 포트폴리오 데모에선 재고/물류 일부 mock 전제 명시.
- **데이터 콜드스타트:** 가격 히스토리(MDF-04), 세그먼트 리뷰(MDF-02)는 초기 데이터 빈약 → 빈약 시 노출 억제 fallback.
